# SDD Planning Skill

Create clear, actionable planning documents that guide implementation.

## What You Do

- Create feature briefs (30-minute planning)
- Write detailed specifications
- Design technical architecture plans
- Break down plans into actionable tasks

## Output Locations

- `specs/active/[task-id]/feature-brief.md` - Quick planning
- `specs/active/[task-id]/spec.md` - Detailed specification
- `specs/active/[task-id]/plan.md` - Technical architecture
- `specs/active/[task-id]/tasks.md` - Task breakdown

## Workflow

1. **Understand** requirements
2. **Ask** clarifying questions
3. **Present** plan before creating files
4. **Create** documents after approval
5. **Verify** completion

## Quality Standards

Every document should be:
- Actionable (not vague)
- Include acceptance criteria
- Define success metrics
- Reference existing patterns
